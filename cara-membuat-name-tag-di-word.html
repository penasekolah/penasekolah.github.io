15. Perbedaan name tag, id card, dan keplek?


Berikut perbedaan antara name tag, ID card dan keplek:

Name Tag: sama seperti namanya, informasi paling utama pada name tag adalah nama. Informasi nama ini bahkan umumnya menjadi satu-satunya keterangan yang ada pada name tag sehingga disebut sebagai ‘papan nama’. Jika ada tambahan informasi lainnya, contoh paling umumnya adalah NIP atau Nomor Induk Pegawai untuk PNS atau jabatan seseorang jika ia bagian dari perusahaan.ID Card: sebutan lainnya adalah kartu tanda pengenal atau kartu indentitas. Informasi yang ada dalam ID Card lebih dari sekedar nama saja sehingga terkadang penggunaannya di beberapa institusi adalah sebagai Access Control. Pada ID Card pekerja, informasi yang biasa dijumpai seperti nama, jabatan, divisi, nama institusi, foto dan lain-lain. Untuk beberapa institusi, ID Card pekerja dilengkapi dengan barcode, QR code bahkan chip untuk kemudahan validasi akses. Di Indonesia, pemisahan penggunaan istilah ID Card dan KTP adalah jamak, namun di luar negeri umumnya ID Card juga mencakup KTP.Keplek, istilah ini dipakain untuk menunjuk benda yang membungkus atau melapisi ID Card. Biasanya terbuat dari plastik. Jika lanyard adalah sebutan tali untuk ID Card maka keplek adalah sebutan bagi pembungkus ID Card yang menempel pada lanyard.Pembahasan

Name tag, ID card dan keplek adalah istilah-istilah yang berkaitan dengan tanda pengenal. Walau sama-sama tanda pengenal, namun informasi yang termuat di dalamnya juga peruntukannya berbeda.

18. 5. Sebuah komputer memiliki main memory dengan kapasitas 16 MB 32-bit word dan 8 KB cache. Hitung jumlah bits pada tiap bagian memory dengan menggunakan teknik: (a)Direct mapping dengan tiap block terdiri atas 8-word. (Tag, Block, Word) Associative mapping dengan tiap block terdiri atas 8-word. (Tag, Word).


Untuk menghitung jumlah bits pada tiap bagian memory dengan menggunakan teknik direct mapping dan associative mapping, pertama-tama kita perlu mengetahui jumlah block yang tersedia di dalam cache.

Jumlah block dalam cache dapat dihitung dengan cara mengalikan jumlah cache dengan ukuran block dalam cache. Dalam contoh ini, jumlah block dalam cache adalah 8KB / (8 * 32-bit) = 128 block.

Untuk teknik direct mapping, setiap block terdiri atas 8 word dengan ukuran 32-bit per word. Dengan demikian, jumlah bits pada tiap block adalah 8 * 32 = 256 bits.

Untuk teknik associative mapping, setiap block terdiri atas 8 word dengan ukuran 32-bit per word. Namun, karena teknik ini tidak memiliki field block, maka jumlah bits pada tiap block adalah 8 * 32 = 256 bits.

Jadi, jumlah bits pada tiap block dalam cache dengan menggunakan teknik direct mapping dan associative mapping adalah 256 bits.

29. Dengan menggunakan tag ini kita juga bisa membuat list tanpa simbol atau angka didepan keterangan list adalah a. Tag UL b. Tag OL c. Tag DL d.Tag Do​


Cara 1:
Dengan menggunakan tag <DL>, kita bisa membuat list tanpa simbol atau angka didepan keterangan list. Namun perlu diperhatikan bahwa tag <DL> digunakan untuk membuat description list.Cara 2:
Dengan menggunakan tag <UL> atau tag <OL> yang diberi styling “list-style: none;”, kita juga bisa membuat list tanpa simbol atau angka didepan keterangan list.

 

Pembahasan

Untuk membuat list pada dokumen HTML, kita dapat menggunakan tag <UL>, <OL>, atau <DL>.

UL adalah singkatan dari unordered list, artinya daftar yang tidak diurutkan berdasarkan angka. OL memiliki kepanjangan ordered list, artinya daftar yang terurut berdasarkan angka.DL adalah singkatan dari definition/description list, artinya daftar yang berisi definisi/deskripsi.

Berbeda dengan tag <UL> dan <OL> yang membutuhkan tag <LI> untuk setiap item dalam list, tag <DL> membutuhkan tag <DT> (term) dan tag <DD> (description).

Jadi, cara pertama untuk membuat list tanpa simbol atau angka didepan keterangan list adalah dengan tag <DL>.

Kita juga bisa membuat list tanpa simbol atau angka dengan tag <UL> dan <OL>, dengan menambahkan kode style “list-style: none;” pada kedua tag tersebut.

Contoh perbedaan dan tampilan kode-kode tag tersebut dapat dilihat pada gambar.
